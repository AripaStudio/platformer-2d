## ۴. گیم‌پلی (Gameplay)

### ۴.۱ ژانر و سبک ✅
- **ژانر اصلی:** پلتفرمر ۲بعدی
- **دیدگاه دوربین:** Side-scrolling (جانبی)

### ۴.۲ مکانیک‌های اصلی ✅

**تصمیم:** ترکیب هر سه سبک — پلتفرمر / پازل / اکشن

| مکانیک | وضعیت | توضیح |
|--------|--------|-------|
| حرکت پایه (راه رفتن، پریدن) | ✅ | هسته اصلی — باید روان و responsive باشه |
| دوبل‌جامپ | ✅ | ۱۴ رای |
| داش (زمینی + هوایی) | ✅ | ۱۱ رای |
| دیوارپریدن / دیواراسلاید | ✅ | ۹ رای |
| پازل محیطی | ✅ | مکانیک حل مسئله درون محیط |
| مبارزه / اکشن | ✅ | دشمن و سیستم مبارزه داریم |
| سیستم سلامت (HP) | ✅ | چند ضربه با نوار HP — ۶ رای |
| سیستم مرگ | ✅ | برگشت به آخرین Checkpoint — ۱۱ رای |


### ۴.۳ حلقه اصلی بازی (Core Loop)
```
ورود به مرحله → حرکت و غلبه بر موانع (پلتفرمر)
             → حل پازل محیطی (پازل)
             → مقابله با دشمنان (اکشن)
             → رسیدن به هدف مرحله → Checkpoint → مرحله بعد
```

### ۴.۴ تعداد بازیکن ✅

**تصمیم:** تک‌نفره با حالت Co-op آنلاین اختیاری

**پیامدهای فنی که باید از ابتدا در نظر گرفته بشن:**
- معماری کد باید از ابتدا **multiplayer-ready** باشه — اضافه کردن بعدی خیلی سخت‌تره
- استفاده از **Godot High-Level Multiplayer API** توصیه می‌شه
- منطق بازی باید از **presentation** جدا باشه تا sync راحت‌تر بشه
- حالت تک‌نفره = همون کد، فقط بدون اتصال شبکه

### ۹.۳ استانداردهای مشارکت
> مشارکت‌کننده‌ها باید **استانداردهای رسمی پروژه Godot** را رعایت کنند:

- **سبک کدنویسی:** [Godot GDScript style guide](https://docs.godotengine.org/en/stable/tutorials/scripting/gdscript/gdscript_styleguide.html)
- **نام‌گذاری:** `snake_case` برای متغیر و تابع، `PascalCase` برای کلاس و نود
- **نام‌گذاری branch:** `feature/dash-mechanic` / `fix/jump-bug`
- **Commit message:** به انگلیسی، فعل در ابتدا — مثال: `Add double jump mechanic`
- **Pull Request:** باید توضیح داشته باشه و حداقل یه نفر review کنه
